# AVRAI BHAM beta Launch Pack

**Date:** March 8, 2026  
**Status:** Draft beta launch package  
**Scope:** Birmingham, Alabama wave-1 closed beta

## Purpose

This pack captures the intended product, runtime, AI2AI, admin, safety, and launch-gate contract for the first Birmingham beta.

This package describes the **beta app/runtime operating model**, not the full post-beta AVRAI OS platformization program.

## Wave-1 Summary

- Geography: Birmingham metro + suburbs
- Initial cohort: 10-25 testers
- Platforms: iPhone + Android on approved devices only
- Age floor: 16+
- Account model: one human, one account, one primary device
- Core stance: offline-first via on-device learning + AI2AI mesh/local relay, with internet as a secondary aid
- Admin stance: secure separate admin app, always-on oversight, no direct human identity exposure by default

## Core Product Promise

AVRAI beta should help Birmingham users:

- find better spots, lists, events, clubs, and communities
- learn who they are through real-world behavior, not phone addiction
- keep working and learning without internet through on-device kernels and AI2AI exchange
- contribute upward to locality and reality-model learning without leaking direct human identity to admin

## Deferred From Wave 1

- business accounts as a first-class role
- expert accounts as a first-class role
- full multi-device continuity
- public release claims beyond Birmingham wave 1

## Required Pre-Phone Smoke Gate

Before any live-phone BHAM beta QA begins, `bham-beta` must have a green baseline simulated smoke run on both platforms:

- `bash work/scripts/proof_run/run_simulated_headless_smoke.sh ios baseline`
- `bash work/scripts/proof_run/run_simulated_headless_smoke.sh android baseline`
