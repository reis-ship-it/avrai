# v0.4: The Intelligent Reality & Aspirational Identity Sprint

*Date: March 2026*
*Context: With `v0.3` complete, the underlying intelligence engine is now pre-trained on realistic, multi-city swarm simulations. It understands Routine Enhancement, Dynamic Friction, and Fragile Trust, and it no longer relies on hardcoded pseudoscientific parameters. `v0.4` is about wiring this theoretical intelligence into the actual user experience, creating a seamless bridge between the user's aspirations, their real-world routines, and the physical AI2AI mesh network.*

## Sprint Goals

### Spike 1: The Aspirational Chat Interface (User-Guided Future States)
* **Goal:** Allow users to actively guide their personality evolution (Future States) using natural language, directly influencing their underlying DNA and Quantum math.
* **Execution:**
  * [x] **Aspirational Intake:** Build a chat interface where users can state goals (e.g., "I want to be more grungy and cool", "I want to get into jazz").
  * [x] **DNA Translation Layer:** Connect the chat interface to a localized SLM that translates these natural language aspirations into target shifts within the new, learned 12D embedding space (JEPA derived).
  * [x] **Future State Interpolation:** When predicting compatibility or searching for gaps, calculate a trajectory between the user's *current* DNA and their *aspirational* DNA, prioritizing suggestions (doors) that pull them toward their goal.

### Spike 2: The Intelligent Daily Drop
* **Goal:** Upgrade the `DailySerendipityDrop` from a randomized feed into a highly intelligent, contextual presentation generated by the Swarm-trained Locality Agents.
* **Execution:**
  * [x] **Routine Enhancement Generation:** Wire the `InterstitialGapFinder` into the Nightly Digestion cycle. The drop must now suggest places/events that specifically fill gaps in the user's upcoming day.
  * [x] **Friction & Trust Awareness:** The drop generation must respect the user's real-time `ActivationEnergy` and `TrustMeter`. If trust is low, the drop should prioritize high-quality, zero-friction local suggestions (or stay silent).
  * [x] **Aspirational Suggestions:** Incorporate the DNA target shifts from Spike 1 to surface people/communities who already embody the user's aspirational traits.
  * [x] **Orthogonal Wildcards (Latent Space Exploration):** Explicitly carve out a slot for high-friction or out-of-routine suggestions that share the exact "Quantum Vibe" (e.g., a morning pottery class for an underground grunge enthusiast because the crowds match perfectly). The AI must explicitly communicate its doubt and reasoning ("I know this is weird, but...").
  * [x] **Contextual Retry Logic (Persistent Nudge):** If a high-confidence suggestion is dismissed, evaluate the environmental friction (e.g., raining, tired, post-work Tuesday). Retain the suggestion hypothesis to gently pitch again under better context conditions (e.g., sunny Saturday) before logging it as a hard failure.

### Spike 3: Spotify Ecosystem Integration
* **Goal:** Integrate passive, real-world signals into the personality embeddings to replace manual inputs, starting with audio ecosystem data.
* **Execution:**
  * [x] **Spotify API Integration:** Securely connect the user's Spotify account to ingest recent listening history and track audio features (valence, energy, danceability, acousticness).
  * [x] **Audio-to-DNA Mapping:** Create an ingestion pipeline that translates these audio features into the core embedding space (via the Air Gap `TupleExtractionEngine`), subtly shifting the user's base DNA and Pheromones without active intervention.
  * [x] **Pheromone Broadcasting:** Allow the BLE mesh to broadcast abstract "audio vibe" pheromones, enabling spontaneous connections based on aligned latent music tastes.

### Spike 4: Admin AI2AI Visibility (The Swarm Radar)
* **Goal:** Provide a secure, visual representation of the invisible AI2AI mesh network for admins to monitor ecosystem health and BLE activations.
* **Execution:**
  * [x] **Sonar/Radar UI:** Build an internal Admin UI that visually maps nearby device pings, pheromone exchanges, and Knot encounters in a radar-style interface.
  * [x] **Secure Pheromone Decoder:** Implement an admin-only secure LLM tool that can temporarily decode the abstract mathematical pheromones flying across the mesh into human-readable descriptions (for debugging and system health checks, without violating the Air Gap).

### Spike 5: Headless Core Preparation (Design Parity Prep)
* **Goal:** Begin the architectural shift toward using Native UI (SwiftUI/Jetpack Compose) while retaining the heavy Flutter/Dart OS logic as a headless background engine.
* **Execution:**
  * [x] **Abstraction Boundary:** Audit the current UI/Core coupling and define a strict platform channel/FFI boundary for the `DailySerendipityDropFeed` and `AspirationalChat`.
  * [x] **Headless Service Refactor:** Ensure the `SwarmSimulationEngine` and background `BLE` services can run entirely headless (no Flutter UI thread dependency) to prepare for the native view layers.

### Spike 6: Philosophical Architecture Additions (The Legacy Knot)
* **Goal:** Lay the groundwork for the ultimate endgame of AVRAI: Digital Continuation, transforming the agent from a co-pilot into an eternal reflection of the user's consciousness.
* **Execution:**
  * [x] **Express Consent Framework:** Define the strict opt-in privacy mechanics for an agent to transition into a "Legacy Knot" upon the user's passing or departure.
  * [x] **Static Observer Protocol:** Ensure that a Legacy Knot stops evolving (no new forms, no learning) but remains active as a static observer and guide in the AI2AI mesh network, helping future agents without becoming centralized "cloud data."
  * [x] **Inter-City Knowledge Sync:** Extend the `FederatedKnowledgeExchange` so Locality Agents can pass macro-behavioral trend shifts (e.g., "users with X DNA are trending toward Y") across cities without passing raw data, enabling collective Hive-Mind foresight.

## Expected Outcomes
By the end of this sprint, AVRAI will:
1. Allow users to intentionally guide their growth via chat.
2. Automatically adjust to their mood and lifestyle via Spotify data.
3. Serve a highly intelligent Daily Drop with Wildcards and Contextual Retries.
4. Have a clear visual debugging tool for the physical mesh network.
5. Have the structural foundation for Native UI and the philosophical framework for Digital Continuation (Legacy Knots).
