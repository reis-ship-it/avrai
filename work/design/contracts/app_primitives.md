# App Primitives

These are the only approved baseline UI building blocks for structured AVRAI pages.

## Core Shell

- `AppShellHost`
- `AppPageScaffold`
- `AppSplitPanel`

## Page Structure

- `AppPageHeader`
- `AppSection`
- `AppSurface`

## Actions

- `AppButtonPrimary`
- `AppButtonSecondary`
- `AppButtonTertiary`
- `AppIconButton`

## Status and Feedback

- `AppInfoBanner`
- `AppStatusBanner`
- `AppStatusPill`
- `AppEmptyState`
- `AppLoadingState`
- `AppErrorState`

## Lists, Rows, and Forms

- `AppListRow`
- `AppSettingRow`
- `AppMetricRow`
- `AppToggleRow`
- `AppFilterChip`
- `AppSearchField`
- `AppFormField`

## Rules

- No structured page may invent its own visual system.
- Decorative glass, glow, neon, or portal styling is not part of the baseline system.
- Semantic colors are reserved for warnings, errors, success, focus, and selection.
- Monochrome hierarchy should come from layout, spacing, type, and surface contrast.
