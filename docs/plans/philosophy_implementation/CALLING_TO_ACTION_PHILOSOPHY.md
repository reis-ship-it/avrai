## 🎯 **The Calling Mechanism**

### **How Users Are Called to Action:**

1. **The World Model Understands the User**
   - Full state encoding: personality (12 dimensions), quantum vibe signature, behavioral patterns, temporal context, trajectory history
   - All of this learned on-device from real behavior -- no required onboarding, no required chat
   - New users get intelligent pre-seeded model weights trained on real personality data (Big Five OCEAN → AVRAI 12 dimensions from 100k+ real examples)

2. **The World Model Understands the Opportunities**
   - Spot quantum states (from business public data, owner curation, visitor behavior)
   - Event quantum states (from organizers, attendee behavior)
   - Community quantum states (from network patterns)
   - List quantum states (from curators and list followers)
   - Business partnerships and sponsorships as first-class doors

3. **The Energy Function Scores Every Door**
   - User state × Door state → Energy score (lower = better fit)
   - Learned from real outcomes, not hardcoded thresholds
   - The original "≥70% compatibility threshold" was a starting heuristic. The energy function learns the right threshold for each user, each type of door, each context. Some users are adventurous (lower threshold for new doors). Some are selective (higher threshold for quality).

4. **The MPC Planner Plans Door Sequences**
   - Not just "here's a good spot" but "this coffee shop → this writers' group → this open mic → this community"
   - Plans trajectories of doors, not one-shot recommendations
   - Active uncertainty reduction: when the model is unsure about you, it shows doors that teach it the most
   - Users are "called" to action at the right time, in the right context

5. **The World Model Learns from Results**
   - What worked? The episodic memory stores outcomes
   - What didn't? Dismissals and non-returns are equally informative
   - Taste drift detection: the model notices when you're changing
   - Seasonal patterns: the model learns your rhythms
   - Continuous improvement: every interaction makes the model better

## 📐 **Mathematical Foundation**

### **The Calling Mechanism (World Model Implementation):**

The "Calling Score" is now implemented by the world model's energy function + MPC planner:

```
energy = E(user_state, action) → scalar  (lower = better door)

user_state = StateEncoder(
  personality_12d,       # 12 AVRAI personality dimensions
  quantum_vibe_state,    # Quantum state representation
  knot_invariants,       # Topological identity
  behavioral_trajectory, # Recent actions and patterns
  temporal_context,      # Time of day, day of week, season
  location_context       # Where they are and where they've been
)

action = any door being considered
  (visit this spot, attend this event, join this community, follow this list, engage with this business)

MPC planner finds the best SEQUENCE of actions:
  actions* = argmin Σ E(predicted_state_t, action_t) for t = 1..H
  
  Where predicted_state_t = TransitionPredictor(state_{t-1}, action_{t-1})

Users are "called" when the MPC planner identifies a high-value door sequence.
The threshold is learned per-user, not hardcoded.
```

**What changed from the original "Calling Score" formula:** the concept is identical -- the AI evaluates how good each opportunity is for the user and surfaces the best ones. What changed is that the scoring is now LEARNED from real outcomes (energy function trained on episodic memories) instead of computed from static compatibility formulas. This means:
- The system gets better with every interaction
- It captures nuances that no formula could encode
- It adapts to each user's unique definition of "meaningful"
